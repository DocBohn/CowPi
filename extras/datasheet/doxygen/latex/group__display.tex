{\footnotesize\ttfamily typedef void($\ast$ cowpi\+\_\+lcd1602\+\_\+send\+\_\+halfbyte\+\_\+t) (uint8\+\_\+t halfbyte, bool is\+\_\+command)}



Pointer to function that sends a halfbyte to the LCD1602 display. 

The {\ttfamily cowpi\+\_\+lcd1602} utility functions all make use of the {\ttfamily \mbox{\hyperlink{group__display_ga4c164ee952d12e22d3874d2c0c909711}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+halfbyte()}}} function pointer.

Initially, {\ttfamily \mbox{\hyperlink{group__display_ga4c164ee952d12e22d3874d2c0c909711}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+halfbyte()}}} is one of two default implementations, depending on the protocol specified in the argument to {\ttfamily \mbox{\hyperlink{group__configuration_ga0195991b9a0da4e8d6db820b9cf87d2a}{cowpi\+\_\+setup()}}}. The SPI implementation is a portable software-\/only implementation that does not use SPI hardware, nor does it use memory-\/mapped I/O of any form. The I2C implementation currently makes use of the I2C hardware using avr-\/libc macros.

Re-\/implementing this function to use SPI or I2C hardware is a possible part of a memory-\/mapped I/O lab assignment.

SPI\+: Assumes the display module\textquotesingle{}s data-\/in line is connected to the microcontroller\textquotesingle{}s COPI pin (D11 on most Arduino boards), the display module\textquotesingle{}s serial-\/clock line is connected to the microcontroller\textquotesingle{}s SCK pin (D13 on most Arduino boards), and the display module\textquotesingle{}s chip-\/select line is connected to Arduino pin D10.

I2C\+: Assumes the display module\textquotesingle{}s data line is connected to the microcontroller\textquotesingle{}s SDA pin (D18 on most Arduino boards) and the display module\textquotesingle{}s serial-\/clock line is connected to the microcontroller\textquotesingle{}s SCL pin (D19 on most Arduino boards).

Places the MAX7219 in decode mode. 

This function places 0x\+FF in the MAX7219\textquotesingle{}s Decode-\/\+Mode register, causing the display to interpret the data register bits as BCD values and place the corresponding decimal numeral on the digits.

This function should not be called when using an 8x8 LED matrix -- there\textquotesingle{}s no harm in doing so, but decode mode really is meant for 7-\/segment displays.

{\footnotesize\ttfamily void cowpi\+\_\+max7219\+\_\+send (\begin{DoxyParamCaption}\item[{uint8\+\_\+t}]{address,  }\item[{uint8\+\_\+t}]{data }\end{DoxyParamCaption})}



Sends a byte of data to be placed in the specified MAX7219 register. 

The firmware program should only need to send data to addresses 1-\/8, the digit registers. Most other registers are handled by specific {\ttfamily cowpi\+\_\+max7219} functions.

This is a portable software-\/only implementation; it does not use SPI hardware, nor does it use memory-\/mapped I/O of any form. Re-\/implementing this function (using a different function name) to use SPI hardware is a possible part of a memory-\/mapped I/O lab assignment.

\begin{DoxyAttention}{Attention}
A re-\/implementation that uses the SPI hardware must invoke {\ttfamily cowpi\+\_\+spi\+\_\+enable} at the start of the function and {\ttfamily cowpi\+\_\+spi\+\_\+disable} at the end.
\end{DoxyAttention}
The necessity to enable and disable the SPI hardware is because on most Arduino boards, the external LED is on the pin that also serves as the CIPO pin. When enabled, the SPI hardware forces the CIPO pin to be an input pin.

Assumes the display module\textquotesingle{}s data-\/in line is connected to the microcontroller\textquotesingle{}s COPI pin (D11 on most Arduino boards), the display module\textquotesingle{}s serial-\/clock line is connected to the microcontroller\textquotesingle{}s SCK pin (D13 on most Arduino boards), and the display module\textquotesingle{}s chip-\/select line is connected to Arduino pin D10.

Pointer to function that sends a halfbyte to the LCD1602 display.

The {\ttfamily cowpi\+\_\+lcd1602} utility functions all make use of the {\ttfamily \mbox{\hyperlink{group__display_ga4c164ee952d12e22d3874d2c0c909711}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+halfbyte()}}} function pointer.

Initially, {\ttfamily \mbox{\hyperlink{group__display_ga4c164ee952d12e22d3874d2c0c909711}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+halfbyte()}}} is one of two default implementations, depending on the protocol specified in the argument to {\ttfamily \mbox{\hyperlink{group__configuration_ga0195991b9a0da4e8d6db820b9cf87d2a}{cowpi\+\_\+setup()}}}. The SPI implementation is a portable software-\/only implementation that does not use SPI hardware, nor does it use memory-\/mapped I/O of any form. The I2C implementation currently makes use of the I2C hardware using avr-\/libc macros.
