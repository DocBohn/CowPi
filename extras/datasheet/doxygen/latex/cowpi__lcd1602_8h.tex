\doxysubsection{Detailed Description}
Utility functions for 2-\/row/16-\/column display modules with 5x8 LCD characters. 

\begin{DoxyAuthor}{Author}
Christopher A. Bohn
\end{DoxyAuthor}
If {\ttfamily LCD1602} is part of the argument to {\ttfamily \mbox{\hyperlink{group__configuration_ga0195991b9a0da4e8d6db820b9cf87d2a}{cowpi\+\_\+setup()}}}, then the display module will be placed in 4-\/bit mode, and\+:
\begin{DoxyItemize}
\item With each character, increment the cursor\textquotesingle{}s location but do not shift the display
\item Turn the display on, the cursor off, and no blinking
\item Clear the display
\end{DoxyItemize}

The LCD1602 is complex enough that the firmware program should call {\ttfamily \mbox{\hyperlink{group__display_ga4304971dec1e44982b41a314f6d7c68c}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+command()}}}, {\ttfamily \mbox{\hyperlink{group__display_ga7c8af4a4dd511977a86fef33b728c8ee}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+character()}}}, or another more-\/specific function. These functions will take care of dividing the data into halfbytes, managing timing, and other complexities, and will call {\ttfamily \mbox{\hyperlink{group__display_ga4c164ee952d12e22d3874d2c0c909711}{cowpi\+\_\+lcd1602\+\_\+send\+\_\+halfbyte()}}}.
